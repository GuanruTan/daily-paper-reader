## 摘要
从人类驾驶演示中学习多样化且高保真度的交通仿真，对于自动驾驶评估至关重要。近期在大语言模型（LLM）中广泛采用的下文预测（NTP）范式已被应用于交通仿真，并通过监督微调（SFT）实现了迭代改进。然而，此类方法限制了对潜在有价值运动 Token 的主动探索，尤其是在次优区域。熵模式为实现由运动 Token 不确定性驱动的探索提供了一个极具前景的视角。受此启发，我们提出了一种新型 Token 化交通仿真策略 R1Sim，这是基于运动 Token 熵模式探索强化学习的初步尝试，并系统分析了不同运动 Token 对仿真结果的影响。具体而言，我们引入了一种熵引导的自适应采样机制，重点关注此前被忽视的、具有高不确定性但高潜力的运动 Token。我们在安全感知奖励设计的引导下，进一步利用群体相对策略优化（GRPO）来优化运动行为。总的来说，这些组件通过多样化的高不确定性采样和群体比较估计，实现了探索与利用之间的平衡权衡，从而产生了真实、安全且多样化的多智能体行为。在 Waymo Sim Agent 基准测试上的大量实验表明，R1Sim 与现有最先进方法相比取得了具有竞争力的性能。

## Abstract
Learning diverse and high-fidelity traffic simulations from human driving demonstrations is crucial for autonomous driving evaluation. The recent next-token prediction (NTP) paradigm, widely adopted in large language models (LLMs), has been applied to traffic simulation and achieves iterative improvements via supervised fine-tuning (SFT). However, such methods limit active exploration of potentially valuable motion tokens, particularly in suboptimal regions. Entropy patterns provide a promising perspective for enabling exploration driven by motion token uncertainty. Motivated by this insight, we propose a novel tokenized traffic simulation policy, R1Sim, which represents an initial attempt to explore reinforcement learning based on motion token entropy patterns, and systematically analyzes the impact of different motion tokens on simulation outcomes. Specifically, we introduce an entropy-guided adaptive sampling mechanism that focuses on previously overlooked motion tokens with high uncertainty yet high potential. We further optimize motion behaviors using Group Relative Policy Optimization (GRPO), guided by a safety-aware reward design. Overall, these components enable a balanced exploration-exploitation trade-off through diverse high-uncertainty sampling and group-wise comparative estimation, resulting in realistic, safe, and diverse multi-agent behaviors. Extensive experiments on the Waymo Sim Agent benchmark demonstrate that R1Sim achieves competitive performance compared to state-of-the-art methods.

#### 1. 核心问题与整体含义（研究动机和背景）
*   **研究背景**：交通仿真是自动驾驶评估的核心。目前最先进的方法（如 SMART, CATK）借鉴了大语言模型（LLM）的“下文预测”（NTP）范式，将智能体轨迹 Token 化并进行自回归建模。
*   **核心问题**：
    1.  **探索受限**：现有模型多采用固定的 Top-K 采样，容易忽略概率虽低但物理上合理且具有潜在价值的“隐藏珠玉”（Hidden Gems）行为。
    2.  **利用偏差**：传统的监督微调（SFT）强迫模型模仿专家数据，但在复杂交互场景下，专家数据可能并非最优，且容易产生协变量偏移。
*   **核心含义**：受 DeepSeek-R1 启发，本论文旨在通过强化学习（RL）平衡交通仿真中的“探索”与“利用”，利用熵（Entropy）模式识别不确定性，并引导模型学习更安全、更真实的驾驶逻辑。

#### 2. 方法论：核心思想与关键技术
*   **核心思想**：引入熵引导的自适应采样来增强探索，并使用群体相对策略优化（GRPO）结合安全感知奖励进行策略精炼。
*   **关键技术细节**：
    *   **熵引导自适应采样（Entropy-Guided Adaptive Sampling）**：计算当前时刻 Token 分布的香农熵 $H_t$。当熵高（不确定性大）时，动态增大采样范围 $K_t$ 以探索更多可能；当熵低（确定性强）时，减小 $K_t$ 以保证稳定性。
    *   **改进的 GRPO 算法**：无需 Critic 网络。通过在同一场景下采样一组（Group）候选 Token 序列，计算组内相对优势（Advantage）。针对交通仿真不稳定的特点，去除了标准差归一化，改用基于均值的奖励归一化。
    *   **安全感知奖励设计**：奖励函数 $r = r_{safe} \times r_{dis}$。其中 $r_{safe}$ 利用分离轴定理（SAT）检测碰撞（碰撞则惩罚），$r_{dis}$ 利用负指数核函数衡量与真实轨迹的贴合度。

#### 3. 实验设计
*   **数据集与场景**：使用 **Waymo Open Motion Dataset (WOMD)**，包含约 48.7 万个训练场景和 4.5 万个测试场景。
*   **Benchmark**：Waymo Sim Agent 官方评价指标，包括真实性元指标（RMM）、运动学似然、交互似然、地图一致性、碰撞率和越野率。
*   **对比方法**：包括 SMART (Tiny/Large), CATK, UniMM, DRoPE, KiGRAS, BehaviorGPT, GUMP 等 SOTA 模型。

#### 4. 资源与算力
*   **硬件**：使用了 **4 张 NVIDIA RTX 4090 GPU**。
*   **训练细节**：
    *   在预训练模型基础上仅微调 **1 个 Epoch**。
    *   总 Batch Size 为 16。
    *   为了提高效率，冻结了地图和智能体编码器，仅训练最后几层。
    *   推理/训练时，每个输入场景生成 32 个候选 Rollout。

#### 5. 实验数量与充分性
*   **实验规模**：在完整的 Waymo 测试集上进行了验证，并针对验证集进行了深入分析。
*   **消融实验**：非常充分。分别验证了：
    1.  不同采样策略（固定 Top-K vs. 熵引导）的影响。
    2.  不同优化算法（IL, SFT, 标准 GRPO, 改进 GRPO）的对比。
    3.  不同奖励函数设计（结果奖励、过程奖励、混合奖励）的效果。
*   **客观性**：通过将数据集划分为“简单”和“困难”子集，客观展示了模型在处理高不确定性场景下的优势。

#### 6. 主要结论与发现
*   **性能提升**：R1Sim 在 RMM 指标上超过了基准模型 SMART 和 CATK，尤其在**碰撞率（安全性）**方面提升显著。
*   **熵的指示作用**：实验证明 Token 熵与场景难度高度相关，高熵场景往往对应低仿真质量，而 R1Sim 能有效缩小高/低难度场景间的熵差距。
*   **探索的价值**：在复杂（高熵）场景中，增加采样多样性并结合 GRPO 比较，能发现比单纯模仿专家数据更安全、更合理的避让行为。

#### 7. 优点（亮点）
*   **范式创新**：首次将 R1 风格的 GRPO 算法引入 Token 化交通仿真领域，摆脱了对 Critic 网络的依赖。
*   **自适应性强**：熵引导采样解决了固定采样参数在简单和复杂场景间难以兼顾的矛盾。
*   **安全性导向**：通过乘法形式的安全奖励设计，使模型在保持轨迹保真度的同时，具备了更强的避障和让行逻辑。

#### 8. 不足与局限
*   **计算开销**：虽然省去了 Critic 网络，但 GRPO 依赖于多组 Rollout 的并行采样，对实时性或内存带宽仍有一定要求。
*   **模型容量限制**：实验中冻结了大部分编码器参数，这可能限制了模型在极端复杂地图结构下的特征提取上限。
*   **泛化性验证**：目前仅在 Waymo 数据集上验证，尚未在其他数据集（如 nuPlan 或真实城市路测数据）上展示跨域泛化能力。
